## Phase Details

### Phase 3: Trial Experience

**Goal:** Users can start a 30-day trial, see remaining time, and when trial expires the tenant moves to sandbox without being blocked.

**Depends on:** Phase 2 (Module-Based Licensing)

**Requirements:** TRIAL-01, TRIAL-02, TRIAL-04

**Success Criteria** (what must be TRUE):

1. User can start a 30-day trial and receive full access to all licensed modules for the trial tier.
2. User sees a countdown or remaining-days UI for the trial (e.g. banner or settings).
3. When trial expires, tenant automatically moves to sandbox tier and can continue using the app (no hard block).

**Plans:** TBD

---

### Phase 4: Payment & Subscription Management

**Goal:** Users can pay for a subscription (new or after trial), convert from trial to paid, and manage their subscription (upgrade/downgrade/cancel); tier changes take effect and billing aligns.

**Depends on:** Phase 3 (Trial Experience)

**Requirements:** PAYMENT-01, PAYMENT-02, TRIAL-03, SUBS-01, SUBS-02

**Success Criteria** (what must be TRUE):

1. System is integrated with a payment gateway (Stripe or Brazilian gateway).
2. User can complete payment for a new subscription or to convert from trial to paid.
3. User can convert from trial to paid through a clear, guided flow.
4. User can view and manage subscription (upgrade, downgrade, cancel) on a dedicated subscription management page.
5. Changing tier takes effect for the tenant and billing reflects the change (aligned with v1.0 licensing).

**Plans:** 3 plans

Plans:
- [ ] 04-01-PLAN.md — Stripe backend: subscriptions table, stripe_events, change_tenant_tier RPC, webhook handler
- [ ] 04-02-PLAN.md — Checkout & trial-to-paid: create-checkout-session, create-billing-portal-session, useSubscription hook, SubscriptionCheckoutFlow, TrialCountdownBanner CTA
- [ ] 04-03-PLAN.md — Subscription management page + i18n (all 4 locales)

### Phase 5: Billing & Invoices

**Goal:** Users can view billing history and obtain invoices (or links to gateway invoices).

**Depends on:** Phase 4 (Payment & Subscription Management)

**Requirements:** BILL-01, BILL-02

**Success Criteria** (what must be TRUE):

1. User can view billing history (past payments and charges).
2. User can obtain invoices, either generated by the system or via link to gateway invoices.

**Plans:** TBD

---

### Phase 6: Trial & Subscription Emails

**Goal:** Automated email sequences support trial reminders and trial expiration warning.

**Depends on:** Phase 3 (Trial Experience; can run in parallel with Phase 4/5 once trial dates exist)

**Requirements:** EMAIL-01, EMAIL-02

**Success Criteria** (what must be TRUE):

1. User receives trial reminder emails (e.g. 7 days, 3 days, 1 day before expiry).
2. User receives a trial expiration warning email when trial ends or on expiry day.
