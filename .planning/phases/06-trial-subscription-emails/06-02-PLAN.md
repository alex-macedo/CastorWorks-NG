  <files>supabase/functions/execute-trial-emails/index.ts</files>
  <action>
Create `supabase/functions/execute-trial-emails/index.ts` following execute-payment-reminders pattern:

1. **Setup**: serve, createClient with SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY. No auth required (cron-invoked like execute-payment-reminders).

2. **Process trial reminders**:
   - SELECT * FROM trial_reminder_due_candidates (view yields tenant_id, reminder_days, recipient_email, trial_ends_at, tenant_name)
   - For each candidate: resolve locale from tenants.settings->>'locale' or default 'en-US'
   - Build HTML via buildTrialReminderHtml(tenant_name, reminder_days, expiryDate, locale, copy)
   - Get subject from copy[locale].reminder_Nd.subject (map reminder_days 7->reminder_7d, etc.)
   - Call sendEmailViaResend(RESEND_API_KEY, recipient_email, subject, html)
   - INSERT into trial_email_logs (tenant_id, email_type, recipient_email, status: ok?'sent':'failed')

3. **Process expiration queue**:
   - SELECT * FROM trial_expiration_email_queue WHERE processed_at IS NULL
   - For each: fetch tenant (must check opt_out_trial_emails — if true, skip and mark processed_at=now())
   - Resolve recipient: COALESCE(tenant.primary_contact_email, owner email from tenant_users+user_profiles where is_owner)
   - If no recipient, mark processed and skip
   - Build HTML via buildTrialExpirationHtml(tenant_name, locale, copy)
   - Send via sendEmailViaResend
   - INSERT into trial_email_logs
   - UPDATE trial_expiration_email_queue SET processed_at = now() WHERE id = ?

4. **Response**: JSON { remindersSent, expirationSent, errors? }
  </action>
