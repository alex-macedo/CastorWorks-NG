<task type="auto">
  <name>Task 1: Tenant trial data — hook or TenantContext</name>
  <files>src/hooks/useTenantTrial.ts and/or src/contexts/TenantContext.tsx</files>
  <action>
Option A — New hook useTenantTrial():  
- Uses useTenantId(). When tenantId is set, query tenants: select trial_ends_at, subscription_tier_id where id = tenantId. RLS allows SELECT for tenant members.  
- Return { trial_ends_at: string | null, subscription_tier_id: string | null, daysRemaining: number | null, isOnTrial: boolean }.  
- daysRemaining = trial_ends_at ? max(0, ceil((new Date(trial_ends_at) - Date.now()) / 86400000)) : null.  
- isOnTrial = subscription_tier_id === 'trial' && trial_ends_at && new Date(trial_ends_at) > new Date().  
- Use TanStack Query key ['tenant', tenantId, 'trial'] so it invalidates with tenant switch.

Option B — Extend TenantContext:  
- In refreshTenants or a separate fetch for "current tenant details", when tenantId is set fetch tenants(id, name, slug, trial_ends_at, subscription_tier_id) for the current tenantId (single row). Expose trial_ends_at, subscription_tier_id (and derived daysRemaining, isOnTrial) from context.

Choose one; recommend Option A (useTenantTrial) to avoid loading trial data for all tenants in the list and keep TenantContext focused on list + current id.
  </action>

  </verify>
