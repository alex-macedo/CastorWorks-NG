<decisions_made>

- **Tailwind v4**: Switched from `@tailwindcss/postcss` (PostCSS plugin) to `@tailwindcss/vite` (Vite plugin) — only way to get theme utilities (`bg-card`, `text-card-foreground`, etc.) emitted in the build.
- **`@theme inline` position**: Must appear *before* `@import "tailwindcss"` in index.css for v4 to pick up custom tokens.
- **SidebarHeaderShell gradient**: Used inline `style` with `linear-gradient()` instead of Tailwind classes to support 3+ colour stops (Tailwind only supports one `via-` stop natively).
- **Role-based header colours**:
  - Default (admin/internal): `#000 → #111827 → #1e2a3a → #253347` (dark + subtle blue tint matching screenshot)
  - Architect: `#052e16 → #064e3b → #065f46 → #047857` (dark forest green → emerald)
  - Client: `#0c1445 → #1e3a8a → #1d4ed8 → #2563eb` (dark navy → blue)
- **Trial lazy expiry**: No cron; get_tenant_licensed_modules flips expired trials to sandbox on every module resolution call. Simple and reliable.
- **Debug session**: `theme-dark-mode-global` archived as `resolved-*`.

</decisions_made>
