  <action>
Create a single migration file that implements:

1. **tenants columns** (ADD COLUMN IF NOT EXISTS):
   - `primary_contact_email TEXT` — configurable primary contact; when set, use instead of owner email (per 06-CONTEXT)
   - `opt_out_trial_emails BOOLEAN NOT NULL DEFAULT false` — tenant-level opt-out; must be respected before sending any trial reminder or expiration email

2. **trial_email_logs table**: audit trail for trial emails sent
   - id UUID PK, tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
   - email_type TEXT NOT NULL CHECK (email_type IN ('reminder_7d','reminder_3d','reminder_1d','expiration'))
   - recipient_email TEXT NOT NULL
   - sent_at TIMESTAMPTZ NOT NULL DEFAULT now()
   - status TEXT NOT NULL DEFAULT 'sent' CHECK (status IN ('sent','failed'))
   - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
   - Index on (tenant_id, email_type)
   - RLS: service_role or authenticated read-own-tenant; GRANT SELECT to authenticated, INSERT to service_role

3. **trial_reminder_due_candidates view** (similar to payment_reminder_due_candidates):
   - Join tenants (subscription_tier_id = 'trial', trial_ends_at IS NOT NULL, COALESCE(opt_out_trial_emails, false) = false)
   - Compute reminder_days (7, 3, or 1) where date(trial_ends_at AT TIME ZONE 'UTC') - reminder_days = date(now() AT TIME ZONE 'UTC')
   - Exclude tenants that already have trial_email_logs row for that (tenant_id, email_type) where email_type = 'reminder_Nd'
   - Include recipient: COALESCE(primary_contact_email, (SELECT up.email FROM tenant_users tu JOIN user_profiles up ON up.user_id = tu.user_id WHERE tu.tenant_id = t.id AND tu.is_owner = true LIMIT 1))
   - Output: tenant_id, reminder_days, recipient_email, trial_ends_at, tenant_name
   - Use a lateral or subquery to get owner email; ensure recipient_email is NOT NULL in output (exclude rows with null recipient)
   - GRANT SELECT to service_role (Edge Function uses service role)

4. **trial_expiration_email_queue table**:
   - id UUID PK, tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
   - created_at TIMESTAMPTZ NOT NULL DEFAULT now()
   - processed_at TIMESTAMPTZ
   - Index on processed_at IS NULL for queue polling
   - RLS/GRANT: service_role for INSERT/SELECT/UPDATE

5. **Trigger on tenants**: AFTER UPDATE FOR EACH ROW
   - WHEN (OLD.subscription_tier_id = 'trial' AND NEW.subscription_tier_id = 'sandbox')
   - EXECUTE: INSERT INTO trial_expiration_email_queue (tenant_id) VALUES (NEW.id)
   - This fires when get_tenant_licensed_modules runs the lazy expiry UPDATE

Use BEGIN/COMMIT. Follow existing migration naming (YYYYMMDD_description.sql). Add COMMENT ON TABLE/VIEW for clarity.
